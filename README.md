# term-ext

Скрипт для разворачивания терминального сетапа одной командой (Linux + macOS).

## Что устанавливается

- **starship** — кастомный prompt
- **fzf** — fuzzy search
- **fzf-tab** — автодополнение через fzf
- **zoxide** — умный `cd`
- **eza** — красивый `ls`
- **bat** — красивый `cat`
- **broot** — файловый менеджер
- **lazydocker** — TUI для Docker

## Два скрипта

| Скрипт | Когда использовать |
|--------|-------------------|
| `install-terminal.sh` | Свой сервер / Mac, ставит системно (sudo / brew) |
| `install-terminal-user.sh` | Общий сервер, без sudo, только для текущего пользователя |

## Установка

### Свой сервер или Mac

```bash
curl -fsSL https://raw.githubusercontent.com/kar43lov/term-ext/main/install-terminal.sh | bash
exec zsh
```

### Общий сервер (без sudo, только для меня)

```bash
curl -fsSL https://raw.githubusercontent.com/kar43lov/term-ext/main/install-terminal-user.sh | bash
exec zsh
```

### Повторный запуск

Скрипт пишет управляемый конфиг в `~/.zshrc.term-ext`, а в `~/.zshrc` добавляет `source`. Повторный запуск обновляет только `~/.zshrc.term-ext` — пользовательские добавки в `~/.zshrc` не затираются.

## Требования

- Linux (x86_64, aarch64) или macOS (Apple Silicon, Intel)
- macOS: Homebrew
- `install-terminal.sh` на Linux: sudo-доступ
- `install-terminal-user.sh` на Linux: только git и curl

## Шпаргалка по командам

### zoxide — умная навигация по папкам

| Команда | Что делает |
|---------|-----------|
| `z foo` | Перейти в папку, содержащую `foo` в пути (из истории посещений) |
| `z foo bar` | Перейти в папку, содержащую `foo` и `bar` |
| `zi` | Интерактивный выбор папки из истории (через fzf) |
| `z -` | Вернуться в предыдущую папку |

zoxide запоминает папки, в которых ты бывал. Чем чаще посещаешь — тем выше приоритет.

### fzf — нечёткий поиск

| Сочетание | Что делает |
|-----------|-----------|
| `Ctrl+R` | Поиск по истории команд |
| `Ctrl+T` | Поиск файлов в текущей папке |
| `Alt+C` | Поиск и переход в папку |

В любом поиске fzf: начни вводить текст для фильтрации, `Enter` — выбрать, `Esc` — отмена.

### fzf-tab — автодополнение

| Действие | Что делает |
|----------|-----------|
| `cd <Tab>` | Выпадающий список папок с превью дерева файлов |
| `git checkout <Tab>` | Выбор ветки через fzf |
| любая команда + `<Tab>` | Автодополнение аргументов через fzf |

### Стрелки — поиск по истории

| Клавиша | Что делает |
|---------|-----------|
| `↑` | Предыдущая команда, начинающаяся с уже введённого текста |
| `↓` | Следующая команда, начинающаяся с уже введённого текста |

Пример: набери `git` и нажми `↑` — будет листать только команды, начинающиеся с `git`.

### eza — красивый ls

| Команда | Что делает |
|---------|-----------|
| `ls` | Список файлов (алиас на `eza`) |
| `ll` | Список с подробностями (`eza -l`) |
| `la` | Все файлы, включая скрытые (`eza -la`) |
| `lt` | Дерево файлов на 2 уровня (`eza --tree --level=2`) |

### bat — красивый cat

| Команда | Что делает |
|---------|-----------|
| `cat файл` | Показать файл с подсветкой синтаксиса (алиас на `bat`) |

### broot — файловый менеджер

| Команда | Что делает |
|---------|-----------|
| `br` | Открыть файловый менеджер в текущей папке |
| `br /path` | Открыть файловый менеджер в указанной папке |

Внутри broot:
- Начни вводить текст — фильтрация файлов
- `Enter` — перейти в папку / открыть файл
- `Ctrl+E` — открыть файл в редакторе
- `Ctrl+T` — открыть терминал в текущей папке
- `Esc` / `Ctrl+Q` — выйти

### Docker — управление контейнерами

| Команда | Что делает |
|---------|-----------|
| `ld` | Открыть lazydocker (TUI для Docker) |
| `dps` | Список контейнеров (аналог `docker ps -a`) |
| `dr` | Restart контейнера (выбор через fzf) |
| `ds` | Stop контейнера |
| `dst` | Start контейнера |
| `drm` | Удалить контейнер (`rm -f`) |
| `dlogs` | Логи контейнера в реальном времени |
| `dr имя` | Restart конкретного контейнера без fzf |

Автоопределение sudo: если `docker ps` работает без sudo — команды без sudo, иначе с sudo.

### Doznanie — запуск контейнеров

Все команды интерактивные: выбор ветки через fzf (`test`, `main`, `feature/dzn-...`), ввод pipeline ID.

#### Main сервис

| Команда | Среда | Контейнер | Порт |
|---------|-------|-----------|------|
| `dozrun` | dev | `doznanie_YYMMDD` | `1MMDD` |
| `dozrun-prod` | prod | `doznanie_YYMMDD` | `1MMDD` |

```bash
dozrun              # интерактивно: ветка → pipeline
dozrun test 32068   # напрямую
```

#### Kafka consumer

| Команда | Среда | Контейнер | Порт |
|---------|-------|-----------|------|
| `dozrun-kafka` | dev | `doznanie__consumer_YYMMDD` | `4MMDD` |
| `dozrun-kafka-prod` | prod | `doznanie__consumer_YYMMDD` | `6MMDD` |

#### ГОСТ-подпись

| Команда | Среда | Контейнер | Порт |
|---------|-------|-----------|------|
| `dozrun-gost` | dev | `doznanie__gost-signer_YYMMDD` | `1MMDD` |
| `dozrun-gost-prod` | prod | `doznanie__gost-signer_YYMMDD` | `2MMDD` |

ГОСТ-команды дополнительно запрашивают (с дефолтами, Enter для подтверждения):
- `CONTAINER_PASSWORD` (по умолчанию `1234567890`)
- `KEY_CONTAINER_NAME` (по умолчанию `81ab0001.000`)
- `CRYPTO_LICENSE` (только prod, по умолчанию `40000-A0000-B0000-C0000-D0000`)

Если контейнер с таким именем уже существует — спросит, удалить ли.

### starship — prompt

Prompt автоматически показывает:
- текущую папку
- ветку git (`🌱 main`)
- статус git (изменённые файлы)
- время выполнения долгих команд (`⏱ 5s`)
- `➜` при успехе, `✗` при ошибке предыдущей команды
